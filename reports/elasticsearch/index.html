Metrics Explanation / 메트릭 설명

  • COLD_START_MS: Elasticsearch HTTP ready time (JVM start to HTTP responding) / ES 시작부터 HTTP 응답까지 시간 (낮을수록 좋음)
  • SEQUENTIAL_INDEX_100_MS: Time to index 100 documents one by one / 100개 문서 순차 인덱싱 시간 (낮을수록 좋음)
  • BULK_INDEX_1000_MS: Time to bulk index 1000 documents in one request / 1000개 문서 벌크 인덱싱 시간 (낮을수록 좋음)
  • SEARCH_MATCH_ALL_AVG_MS: Average match_all query latency (10 runs) / match_all 쿼리 평균 지연시간 (낮을수록 좋음)
  • SEARCH_TERM_AVG_MS: Average term query latency (10 runs) / term 쿼리 평균 지연시간 (낮을수록 좋음)

Test Methodology / 테스트 방법론

  • 환경: Kubernetes Job으로 각 인스턴스 타입에서 5회 반복 실행 (pod anti-affinity로 노드 격리 보장)
  • ES 설정: Elasticsearch 8.11.0, JVM Heap 60% (MaxRAMPercentage), G1GC, single-node 모드
  • Cold Start 측정: JVM 시작부터 ES HTTP 포트(9200)가 응답할 때까지의 시간
  • Index 테스트: 순차 인덱싱 100개 문서 (개별 PUT), 벌크 인덱싱 1000개 문서 (단일 _bulk 요청)
  • Search 테스트: match_all과 term 쿼리를 각각 10회 실행 후 평균 latency 측정
  • 격리: 각 테스트는 전용 노드에서 실행되며, 다른 벤치마크 워크로드와 동시 실행 방지
