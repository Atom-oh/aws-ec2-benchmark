테스트 방법론

왜 Nginx 벤치마크가 중요한가?

  • 웹 서버 성능 - 실제 HTTP 요청 처리 능력 측정
  • 동시성 처리 - 동시 연결 처리 능력
  • 지연 시간 - 사용자 경험에 직접적 영향

수집 메트릭

메트릭설명방향

Latency평균 응답 지연 시간낮을수록 좋음

테스트 설정

테스트 격리 보장

  • Pod Anti-affinity: 각 벤치마크가 전용 노드에서 단독 실행
  • Karpenter: 인스턴스 타입별 새 노드 동적 프로비저닝
