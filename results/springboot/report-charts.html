1. Test Methodology

Application: Spring Boot 3.2 + Java 21 minimal REST API (spring-boot-simple) for throughput, Spring PetClinic for cold start.

Load Tool: wrk (2 threads) — 30s warm-up, then 3 phases: 50 conn (60s), 100 conn (60s), 200 conn (30s). Each instance measured 5 times, median reported.

JVM Tuning: -XX:MaxRAMPercentage=60.0 -XX:+UseG1GC -XX:MaxGCPauseMillis=100

Architecture TieredCompilation JIT Strategy Rationale
Intel AMD ON (default) C1 → C2 (staged compilation) x86 JIT는 C1+C2 조합이 최적. C1으로 빠르게 워밍업 후 C2가 최종 최적화.
Graviton OFF (-XX:-TieredCompilation) C2 only (direct optimization) ARM C2 컴파일러가 생성하는 네이티브 코드가 C1 없이 직접 실행 시 더 효율적. 30s warm-up이 C2 지연을 상쇄.

Note: c7i-flex.xlarge는 클러스터 AZ 제약으로 C2 only 결과를 사용합니다 (C1+C2 대비 소폭 차이 예상).
