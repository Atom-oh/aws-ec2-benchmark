iperf3 Network 벤치마크

네트워크 대역폭 및 지연시간 | 54개 EC2 인스턴스 타입 비교

리전: ap-northeast-2 (서울) | 인스턴스 크기: xlarge (4 vCPU)

목차

테스트 방법론

왜 iperf3 벤치마크가 중요한가?

  • 실제 네트워크 성능 측정 - 인스턴스 유형별 실제 네트워크 처리량 비교
  • 고성능 네트워킹 옵션 비교 - 'n' 접미사 인스턴스(예: c6in)의 네트워크 향상 효과 확인
  • 비용 최적화 - 네트워크 집약적 워크로드에 적합한 인스턴스 선택

수집 메트릭

메트릭설명단위방향
TCP Single Stream단일 TCP 연결 대역폭Gbps높을수록 좋음
TCP Parallel (8 streams)8개 병렬 TCP 연결 총 대역폭Gbps높을수록 좋음
TCP Reverse역방향 TCP 대역폭Gbps높을수록 좋음
UDP BandwidthUDP 대역폭Mbps높을수록 좋음
Jitter패킷 지연 변동ms낮을수록 좋음
Packet Loss패킷 손실률%낮을수록 좋음

테스트 환경

테스트 도구

iperf3 3.x

클러스터

EKS 1.36 / Karpenter

네트워크

Same AZ, VPC CNI

테스트 인스턴스

54개 (Intel/AMD/Graviton)

테스트 구성

설정값설명
TCP Single Stream-c server -t 3030초간 단일 TCP 연결 테스트
TCP Parallel-c server -P 8 -t 308개 병렬 스트림으로 30초 테스트
TCP Reverse-c server -R -t 30역방향(서버→클라이언트) 대역폭
UDP-c server -u -b 1G -t 301Gbps 타겟으로 UDP 테스트

알려진 데이터 결손

c7g/c7i/m8i.xlarge는 원본 로그가 5회 중 1회 "Broken pipe" 오류로 완전 실패해 TCP Single Stream만 4회 평균(다른 지표는 5회 정상).

아키텍처별 세대별 성능 비교

인스턴스 패밀리별 비교 (C/M/R x 아키텍처)

가격 대비 성능 (Cost Efficiency)

용도별 추천 인스턴스

네트워크 요구사항별 최적 인스턴스 가이드

초고속 네트워크 (25 Gbps+)

  • m6idn.xlarge - 최고 대역폭
  • m6in.xlarge, c6in.xlarge
  • 용도: 빅데이터, 분산 스토리지, HPC

범용 네트워크 (~12 Gbps)

  • c7i.xlarge - 최신 Intel 7세대
  • m8g.xlarge - Graviton4 최신
  • 용도: 일반 웹서비스, API 서버

저지터 (실시간 통신)

  • 지터 Top 20 차트 참고 (성능 분석 섹션)
  • 용도: 실시간 음성/영상, 게임

피해야 할 인스턴스

  • 병렬 대역폭 하위권(가격 대비 성능 섹션 참고)
  • 이유: 네트워크 대역폭 제한 심함

전체 결과

결론

  • 네트워크 최적화 인스턴스('n' 접미사)가 압도적으로 높은 병렬 대역폭 제공
  • Graviton4(8세대)는 네트워크 측면에서는 Intel 대비 뚜렷한 우위 없음
